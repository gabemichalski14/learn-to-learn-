# Curriculum Coverage Rubric — the standing gap-scanner

**Purpose.** A reading curriculum has a small number of evidence-validated
components. This file maps **every** one of them to **where we cover it** and
**how well** (✅ well / ⚠️ partial / ❌ missing / ⬜ deliberate scope boundary).
Re-run it **every time we add a level, a game, or new research lands** — a gap
can't hide if every component is on one checklist.

> Built from a 4-domain literature dive (reading-science frameworks · dyslexia /
> Orton-Gillingham · fluency-vocabulary-comprehension · cognitive & motivational
> psychology). All four converged on the same gaps. Sources at the bottom.
> *Not clinical/legal advice — validate with a reading specialist.*

## The honest one-line verdict

We have a **near-exemplary _Word Recognition_ strand** (the "D" in the Simple
View) delivered with strong structured-literacy fidelity — and we currently leave
the **_Language Comprehension_ strand near zero**, plus three real gaps **inside
our own decoding mission** (connected text, fluency, heart words) and a missing
**memory architecture** (spaced cumulative review + interleaving). Because
`Reading Comprehension = Decoding × Language Comprehension` is **multiplicative**,
a child can ace every game and still not comprehend — so these are real, not
cosmetic.

## B. The gaps, ranked (what to build, in order)

1. **Connected decodable text** — the bridge itself; the substrate fluency + orthographic mapping both need. Highest ROI; requires no new linguistic content (string taught GPCs into sentences). Use **decodable**, never predictable/leveled (3-cueing trains guessing).
2. **Real fluency** — phrase → sentence → short-passage **repeated reading** with **model-then-echo prosody**; reframe "read it fast" → "read it smooth/expressive." WCPM is a *dashboard* metric, **never** shown to the child as a target.
3. **Heart words / orthographic mapping of irregular high-frequency words** — small, bounded, huge payoff (these words saturate every sentence). Routine: hear → segment → map sounds to graphemes → flag the "tricky" part → spell back. Reuses our PA + encoding strengths.
4. **Memory architecture: spaced cumulative review + interleaving** — the two strongest effects in learning science, currently absent. Build a per-skill review queue (fixed ~weekly-feel intervals, "less is more" for young kids), feed it our captured confusions, interleave confusable pairs *after* blocked acquisition.
5. **Seed morphology earlier** (L2–L4: compounds, -s/-ing, simple prefixes) — keeps formal roots at L9–L10.
6. **Close the confusion-data loop** (auto-generate targeted review/interleaving) · 7. **At-risk screener** (RAN/PA) · 8. **ADHD session layer**.

## C. The one strategic decision (Language Comprehension) — owner's call

By the Simple View, we build **D** and leave **LC ≈ 0**. Two honest, defensible paths — pick deliberately:

- **Own the boundary (lower effort, fully legitimate):** position the product as the **Word Recognition strand** — "we teach kids to *lift words off the page*; pair us with read-alouds & conversation for the rest." For a **dyslexic-profile** child whose oral language is intact, decoding *is* the bottleneck, so this focus is exactly right. **Just never claim a standalone "non-reader → fluent comprehending reader" outcome** while teaching only one factor of a product.
- **Add a light LC layer (more work):** vocabulary first (cheap — reframe affix/root levels into a meaning engine + Tier-2 "word garden"), then **sentence-level** comprehension once connected text exists, with knowledge woven via the existing narrative/read-aloud system. **Avoid** standalone "find the main idea / inference" minigames — the evidence (Hirsch/Wexler) says content-free strategy drills don't transfer.

**This is the single decision that should gate further level-building.**

## D. How to re-scan (the standing process)

1. **On every new level or game:** re-score Table A. A new level must not regress any ✅, and should state which ⚠️/❌ it advances.
2. **When new research lands:** add the component/row + source; re-rank §B.
3. **Guardrails to keep us honest:** the no-shame / no-speed-pressure rule (fluency = "smooth," not stopwatch); decodable (not predictable) text; multisensory ≠ learning-styles; block-before-interleave.
4. Treat this file as the **definition of "complete"** — building features that aren't on it, while ❌ rows remain, is how a curriculum drifts.
